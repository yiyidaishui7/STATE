## Slide 2 — 训练曲线对比（fig2_training_curves.png）

**图：fig2_training_curves.png（2×2 子图）**

### 左上：Training Loss

```
Baseline（蓝）：  24.5 → 20.6（降幅 16%）
STATE+MMD（橙）： 23.9 → 20.0（降幅 16%）
```

- 两条曲线形状相似，说明训练稳定性相当
- MMD 模型训练损失略低（~0.6），可能因为对齐约束隐式减少了过拟合
- step ~900 附近 MMD 曲线有轻微凸起：α 开始增加，对齐损失引入短暂扰动，随后恢复

### 右上：Validation Loss（HepG2 零样本）

```
Baseline：   26.7 → 21.9
STATE+MMD：  26.4 → 21.9
```

- 两者收敛到**完全相同的水平（~21.9）**
- 这是最直接的证据：对 HepG2 的零样本预测能力，两个模型一样

### 左下：MMD & ADV Loss（仅 STATE+MMD）

- **MMD loss（橙）**：从 ~0.03 降至 ~0.01，持续下降，分布距离在缩小 ✅
- **ADV loss（紫）**：稳定在 ~1.099，即 ln(3) = 1.0986（三分类随机猜测的熵值）

  > **ln(3) 的意义：** 若判别器对三个域各输出 1/3 概率，交叉熵 = -3×(1/3)×log(1/3) = ln(3)。ADV loss 稳定在这里，说明判别器**无法比随机猜测更好**地区分 K562/RPE1/Jurkat。GRL 成功击败了对抗判别器。

### 右下：α Schedule（阶梯状）

```
step ~900：  α = 0.2（epoch 5 开始）
step ~1100： α = 0.4
step ~1300： α = 0.6
step ~1500： α = 0.8
step ~1700： α = 1.0（epoch 10，之后保持）
```

- 阶梯状是因为用 `current_epoch`（整数）计算，每个 epoch 内所有 step 的 α 完全相同
- **这正是老师指出的问题**：每个 epoch 边界处 α 跳变，loss 会有不连续的扰动
- 新版 cosine 实现已完成，正在训练中（见后续）

---
**讲稿**

先看训练曲线，这里有几个细节值得说明。

训练损失这里，MMD 模型比 baseline 低了约 0.6。这并不是因为 MMD 帮助了预测，而是因为对齐损失作为一种隐式正则化，让模型泛化更好。但注意验证集（HepG2）上两者完全一样，所以这个训练集上的优势没有转化为零样本能力。

左下的 ADV loss 稳定在 ln(3)，这是一个很好的收敛信号。ln(3)≈1.099 是三分类在完全随机情况下的交叉熵，判别器输出均匀分布时就是这个值。ADV 稳定在这里说明 GRL 从对抗博弈角度成功了——训练中的那个判别器被完全击败。

右下 α 是阶梯状的。可以看到五个台阶，每个 epoch 跳一次。老师指出这个问题是对的，连续的 schedule 更合理，我们已经实现了 cosine 版本，稍后会说。

## Slide 3 — Pearson r 结果（fig1_pearson_bar.png）

**图：fig1_pearson_bar.png**

### 细胞级 Pearson r（Mode A）

评估方法：对每个 HepG2 细胞，用模型预测所有基因的表达分数，与真实表达计算 Pearson 相关系数。

| 模型 | Mean r | Std | Median | N cells |
|------|--------|-----|--------|---------|
| No-MMD Baseline | 0.7003 | 0.0456 | 0.7039 | 2000 |
| STATE+MMD | 0.6986 | 0.0460 | 0.7016 | 2000 |
| **Δ (MMD − Baseline)** | **−0.0017** | — | — | — |

### 统计解读

- Δ = −0.0017，是 std（0.046）的 **1/27**
- 误差棒（±std）完全重叠
- **结论：两者在统计上没有差异（差值在噪声范围内）**

### r = 0.70 本身的意义

- R² ≈ 0.49：模型解释了 HepG2 基因表达方差的约 49%
- 这是**零样本**结果——HepG2 从未参与训练
- 说明多域联合训练（K562+RPE1+Jurkat）本身就产生了强泛化能力

---
**讲稿**

这是最关键的结果图。两根柱子的高度几乎完全一样，差值 0.0017 比标准差小 27 倍。从统计上看，这两个模型对 HepG2 的预测能力没有任何差异。

这个结果直接回答了"MMD 有没有用"的问题：**在零样本预测这个任务上，MMD 没有额外帮助。**

但这不意味着 r=0.70 是个差结果——恰恰相反，零样本能达到 0.70 是非常好的泛化能力，只是这个能力来自多域预训练本身，不是 MMD 带来的。

## Slide 4 — 域对齐指标对比（fig3_alignment_metrics.png）

**图：fig3_alignment_metrics.png（3个子图）**

### 三个指标的含义与结果

**① DomainClsAcc（域分类准确率，越低越好）**

```
Baseline：  99.66%
STATE+MMD： 99.34%
随机基线：  33.3%（1/3）
```

- 训练一个新的 MLP，用 K562/RPE1/Jurkat 的 CLS embedding 来区分三个域
- 33.3% = 完美对齐（随机猜），100% = 完全分离
- 两个模型都在 99%+，说明 embedding 里**仍然保留了强烈的域特异性信息**
- MMD 把这个指标从 99.66% 降到 99.34%，下降了 0.32%，几乎没有变化

**② Silhouette Score（轮廓系数，越接近 0 越好）**

```
Baseline：  0.3965
STATE+MMD： 0.0412
```

- 衡量每个样本"离本域中心"vs"离最近他域中心"的比值
- +1 = 各域完全分离，0 = 完全混合，-1 = 分配错误
- 从 0.40 降到 0.04：**MMD 使三个域在局部几何上几乎完全混合** ✅
- 这是最直观的对齐成功证据

**③ MMD (mean)（分布统计距离，越低越好）**

```
Baseline：  0.1878
STATE+MMD： 0.0009
```

- 直接计算三对域（K562-RPE1、K562-Jurkat、RPE1-Jurkat）embedding 的 MMD 值
- 从 0.1878 降到 0.0009：**降低了约 200 倍** ✅
- MMD 训练目标直接被优化到了接近 0

---
**讲稿**

这三个指标放在一起，形成了一个有意思的对比。

Silhouette 和 MMD 都说对齐成功了：从 UMAP 图和这两个数字来看，三个域的 embedding 在局部几何和统计分布上已经非常接近。

但 DomainClsAcc 几乎没有变化，两个模型都是 99%+。这是"GRL 悖论"——下一页详细解释。

这三个指标同时存在，揭示了一个重要事实：MMD 对齐了分布的低阶统计量，但 embedding 里还藏着高阶的域特异性信息，这些信息对分类器来说仍然清晰可见。

```
- K562（红）：右下角，紧密独立的簇
- RPE1（蓝）：左侧，两个子簇
```

### 右图：STATE+MMD


### 视觉直觉

### 为什么 UMAP 可以混合，但 DomainClsAcc 还是 99%？

UMAP 是 2 维投影，损失了大量高维信息。MLP 分类器在原始 512 维空间里工作，能找到 UMAP 看不见的高阶判别边界。两者并不矛盾。

---
**讲稿**

UMAP 是最直观的展示。左图 baseline 三个域各自聚成一团，右图 MMD 之后三个域混在一起，视觉上非常清晰地展示了对齐效果。

但这里需要特别说明：UMAP 是把 512 维空间压缩到 2 维，这个过程会丢失大量信息。虽然 2 维 UMAP 上看起来混合了，但原始 512 维空间里还有 MLP 能找到的判别特征，所以 DomainClsAcc 还是 99%。这不是矛盾，而是两种工具测量了不同层次的对齐程度。

## Slide 6 — GRL 悖论与深层分析

### 矛盾现象

| 证据 | 说明 |
|------|------|
| ADV loss ≈ ln(3) | 训练时判别器被完全击败 ✅ |
| Silhouette = 0.04 | 局部几何已混合 ✅ |
| MMD = 0.0009 | 统计分布接近相同 ✅ |
| **DomainClsAcc = 99%** | **新 MLP 仍能 99% 区分域** ❌ |

### 根本原因：GRL 只击败了"那一个"判别器

```
训练时：
  encoder → GRL → 判别器A（被 GRL 反转梯度欺骗）
  编码器学会：专门欺骗判别器 A

评估时：
  encoder → 判别器B（全新独立训练，无 GRL）
  编码器没有针对 B 防御 → B 自由分类 → 99% 准确
```

GRL 是一个"对抗博弈"：编码器只学会欺骗训练中遇到的那一个特定判别器，换一个新判别器就失效了。

### 另一个原因：adv_weight = 0.1 太弱

$$\mathcal{L} = \mathcal{L}_{pred} + \alpha \cdot (2.0 \times \mathcal{L}_{MMD} + \underbrace{0.1}_{\text{极小}} \times \mathcal{L}_{adv})$$

- MMD 权重 2.0，ADV 权重 0.1，比值 20:1
- GRL 对 encoder 施加的反向梯度非常弱
- encoder 只需稍微混淆判别器即可满足损失要求，大量域信息被保留

### 三个层次的对齐

```
低阶统计（均值/方差）  → MMD=0.0009   ✅ 完全对齐
局部几何（近邻结构）   → Silhouette=0.04 ✅ 基本对齐
高阶判别特征（非线性） → DomainClsAcc=99% ❌ 未对齐
```

---
**讲稿**

这是最值得深入讨论的现象。训练时 ADV loss 在 ln(3) 说明对抗博弈里的判别器被彻底击败了，但评估时新的 MLP 还是能 99% 区分域。

理解这个的关键是：GRL 训练的是"欺骗特定判别器"的能力，不是"消除所有域信息"的能力。编码器学会了针对训练中的那一个判别器产生让它混淆的输出，但它没有真正把域信息从 embedding 里消除。换一个新的、没有被 GRL 影响过的分类器，它就能自由地找到域的判别特征。

更重要的是 adv_weight=0.1 太小。整个对齐损失里 MMD 占 2.0，ADV 只占 0.1，GRL 对编码器的压力是很弱的。如果把 adv_weight 增大到 0.5 或 1.0，GRL 可能真的能消除更多域信息，但代价是可能影响预测性能。这是一个值得做的消融实验。

## Slide 8 — 核心结论与下一步

### 实验结论

| 问题 | 答案 |
|------|------|
| MMD 能对齐分布吗？ | ✅ 能，Silhouette 0.40→0.04，MMD 0.19→0.001 |
| 对齐能提升预测性能吗？ | ❌ 不能，Δr = −0.0017，统计上无差异 |
| r=0.70 从哪里来？ | 多域联合预训练本身的泛化能力 |
| HepG2 有数据泄露吗？ | ❌ 没有，代码确认 HepG2 只在 val |

### 针对老师三个问题的回应

**Q1：α 是什么？**
α 是对齐损失的权重系数：$\mathcal{L} = \mathcal{L}_{pred} + \alpha(\mathcal{L}_{MMD} + \mathcal{L}_{adv})$
epoch 0–4：α=0（纯预测），epoch 5–9：线性增加，epoch 10+：α=1

**Q2：α 应该是连续的**
已实现 cosine 版本，用 `global_step` 代替 `current_epoch`：
$\alpha = 0.5 \times (1 - \cos(\pi \times \text{progress}))$
cosine 版本正在训练，结果即将出来

**Q3：Baseline 是否有 HepG2 泄露**
已确认无泄露，重新训练了干净的 baseline，r=0.700 是真正的零样本结果

### 正在进行的实验

| 实验 | 目的 | 状态 |
|------|------|------|
| mmd_aae_cosine | 验证连续 α 是否改善训练稳定性 | 训练中 |
| LOO-Jurkat | Jurkat 零样本泛化能力 | 等待 |
| LOO-RPE1 | RPE1 零样本泛化能力 | 等待 |
| LOO-K562 | K562 零样本泛化能力 | 等待 |

LOO 实验将回答：r≈0.70 是对所有细胞系普遍成立，还是 HepG2 特例？

---
**讲稿**

最终结论总结在这张表里。MMD 对分布的低阶统计量对齐是成功的，UMAP 也能看到混合效果，但这没有转化为下游任务的提升。

r=0.70 不是 MMD 带来的，是多域联合训练本身学到的泛化能力。这是一个"负结果"，但是有价值的——它告诉我们，对于这个任务，简单地对齐分布统计量是不够的。

后续有两个方向：一是增大 adv_weight 做消融，看更强的 GRL 是否有帮助；二是 LOO 实验，验证这个泛化能力是否对所有四个细胞系都成立。

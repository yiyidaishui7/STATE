# STATE + MMD-AAE 项目总结与规划

> 单细胞扰动预测中的域对齐与扰动方向评估

---

## 一、项目背景

### 任务定义

给定一个基因敲除扰动（如 TP53 knockout），预测该扰动对细胞基因表达谱的影响——即哪些基因上调、哪些基因下调，以及幅度多大。

### 数据

| 数据集 | 用途 | 细胞数 | 扰动数 |
|--------|------|--------|--------|
| K562 | 训练域 | — | — |
| RPE1 | 训练域 | — | — |
| Jurkat | 训练域 | — | — |
| HepG2 | **零样本验证**（从未出现在训练中） | ~5000 ctrl + 68×N pert | 68 种基因敲除 |

所有数据均为 log1p 归一化基因表达矩阵，形状 `(n_cells, 18080)`。

### 核心假设

如果三个训练域的 CLS embedding 能对齐到同一特征空间，HepG2 细胞可以"搭便车"利用该共享空间，实现零样本迁移。

---

## 二、方法架构

### STATE 基础模型

```
细胞基因表达 (B, 2048)
      ↓  ESM2 蛋白质 embedding 查表 + 线性投影
(B, 2048, 512)  基因 token 序列
      ↓  8 层 Flash Transformer
(B, 2048, 512)  上下文化输出
      ↓  取第 0 位（CLS token）
(B, 512)  CLS embedding
```

### 我们叠加的 MMD-AAE 域对齐模块

```
CLS embedding (B, 512)
      ├──→ binary_decoder → TabularLoss (pred_loss)
      │                   + logFC MSE   (logfc_loss, 路线一新增)
      │
      └──→ GRL (梯度反转) → domain_disc → CrossEntropy (adv_loss)
           + 成对 MMD(K562↔RPE1, K562↔Jurkat, RPE1↔Jurkat)  (mmd_loss)
```

### 训练损失函数

```
total_loss = pred_loss                              # 系数 1.0（隐式）
           + 0.1 × logfc_loss                       # 路线一新增，系数 0.1
           + α × (2.0 × mmd_loss + 0.1 × adv_loss) # MMD-AAE 域对齐
```

**Alpha 预热调度（余弦）：**

| Epoch | Alpha | 说明 |
|-------|-------|------|
| 0 – 4 | 0 | 纯预测训练，对齐模块关闭 |
| 5 – 9 | 0 → 1（余弦） | 缓慢引入对齐 |
| 10 – 15 | 1.0 | 完整域对齐 |

### 关键权重不对等

| 项 | 系数 | 实际量级 | 梯度贡献 |
|----|------|---------|---------|
| TabularLoss | ×1.0 | ~1–5 | **主导** |
| logfc_loss | ×0.1 | ~0.01–0.1 | 极弱，被淹没 |
| mmd_loss | ×2.0α | ~0.001–0.01 | epoch≥5 后存在 |
| adv_loss | ×0.1α | ~0.5–2.0 | 很弱 |

**根本矛盾**：梯度压力 90% 以上来自 TabularLoss，而 TabularLoss 只优化分布形状，完全不奖励扰动方向的正确性。

## 四、评估流程与结果

### 第一轮：全基因组 Pearson r（被否定）

- **方法**：在 HepG2 全部 18,080 个基因上计算 pred vs actual 的 Pearson r
- **结果**：r ≈ **0.70**
- **问题**：误导性指标

```
18,080 个基因 = ~17,800 个不变基因（pred≈0, actual≈0）
              + ~280 个真正变化的 DEG

17,800 对"0 vs 0"贡献了绝大部分相关性 → Pearson r 虚高到 0.70
即使模型什么都没学，只要把所有基因预测成 0 也能得到高 r
```

- **结论**：被老师否定，需改用 DEG 子集评估。

---

### 第二轮：域对齐质量评估（成功）

- **方法**：提取 K562/RPE1/Jurkat/HepG2 的 CLS embedding，计算四个指标
- **脚本**：`scripts/eval_state_domain_alignment.py`

| 指标 | STATE+MMD | 含义 |
|------|-----------|------|
| DomainClsAcc | 99.61% | 判别器仍能分类（paradox，见下） |
| Silhouette Score | **0.037 ≈ 0** | 三域 embedding 空间高度融合 |
| MMD² | **6×10⁻⁴** | 跨域分布距离极小 |
| CORAL | **≈ 0** | 协方差矩阵对齐 |

> **DomainClsAcc=99.61% 悖论解释**：这是在 embedding 上重新训练了一个线性分类器。Silhouette≈0 说明三域在几何上已融合；但域信息仍以非线性方式编码在 embedding 中，线性分类器仍能以高准确率区分。Silhouette 是更可靠的对齐指标。

- **结论**：域对齐在表征空间层面成功，三个细胞系的 CLS embedding 已融合。

---

### 第三轮：DEG Pearson（谢老师方法，初版）

- **方法**：
  1. 对每个扰动，Wilcoxon + BH 校正筛出显著 DEG（adj p < 0.20）
  2. 计算 DEG 子集的真实 logFC（pert_mean − ctrl_mean）
  3. 用模型预测 logFC（decode(CLS_pert) − decode(CLS_ctrl)），取 DEG 子集
  4. 计算 Pearson r
- **脚本**：`scripts/eval_deg_pearson.py`（原版）

| 模型 | Mean DEG Pearson r | 正相关比例 |
|------|-------------------|-----------|
| STATE+MMD | -0.035 | ~48% |
| Baseline | -0.021 | ~48% |
| 随机期望 | ≈ 0 | 50% |

- **初步结论**：两者均约等于随机猜测。

---

### 第四轮：LOO 留一法（排除域泛化假说）

- **方法**：分别留一个细胞系做验证，其余三个训练，检验 DEG Pearson
- **结果**：所有验证集 DEG Pearson ≈ 0（范围 -0.035 ~ +0.013）
- **关键意义**：连训练集细胞系自己都预测不准 → 问题不在"HepG2 是零样本"，而在模型本身

---

### 第五轮：DES 基因重叠率

- **方法**：预测 top-50 基因 vs 真实 top-50 DEG 的交集大小
- **脚本**：`scripts/eval_des.py`

| 模型 | DES 均值 | DES>0 比例 |
|------|---------|-----------|
| STATE+MMD | 0.0054 | 21% |
| Baseline | 0.0042 | — |
| 随机基线 | 0.0028 | — |

- **结论**：与 DEG Pearson 一致，都在随机水平附近，从命中率角度验证同一问题。

---

### 第六轮：Top-k PCC 曲线（新方法，本次新增）

**改进思路**：原方法用 p-value 阈值筛基因，会把大量弱 DEG 纳入计算，稀释信号。改为按 |Wilcoxon scores| 降序选最显著的 top-k 个基因，k 从 1 逐渐增大，观察 PCC 随 k 变化的曲线。

**方法流程：**

```
对每个扰动：
  1. sc.tl.rank_genes_groups(method="wilcoxon")
     → scores 列有正负（正=扰动后该基因相对ctrl上调）
     → 按 |scores| 降序排列全部 18,080 个基因

  2. 编码 ctrl/pert 细胞 → 平均 CLS → decode 全基因
     → pred_logfc_all = pred_pert - pred_ctrl
     → actual_logfc_all = actual_pert - actual_ctrl

  3. for k in [2, 3, ..., 10]:
       取排名前 k 的基因
       PCC_k = pearsonr(pred_logfc[top-k], actual_logfc[top-k])

  输出：一条 PCC 随 k 变化的曲线
```

> k=1 时只有 1 个数据点，pearsonr 无意义（输出 NaN）。
> k=2 时 pearsonr 恒等于 ±1（数学 artifact），数字参考价值有限。

**结果：**

| k | STATE+MMD mean PCC | Baseline mean PCC |
|---|-------------------|------------------|
| 2 | 0.134（artifact）| 0.015（artifact）|
| 3 | **+0.096** | -0.112 |
| 4 | **+0.102** | -0.036 |
| 5 | **+0.126** | -0.021 |
| 6 | **+0.115** | -0.026 |
| 7 | **+0.088** | -0.013 |
| 8 | +0.052 | +0.020 |
| 9 | +0.032 | -0.001 |
| 10 | +0.053 | +0.014 |

**正相关扰动比例（Fraction PCC > 0）：**

| k | STATE+MMD | Baseline |
|---|-----------|----------|
| 3~6 | **~55–63%** | ~42–50% |
| 7~10 | ~55–63% | ~50–62% |
| 随机期望 | 50% | 50% |

## 五、综合分析

### 5.1 各指标对比总结

| 评估方法 | STATE+MMD | Baseline | 结论 |
|---------|-----------|----------|------|
| 全基因组 Pearson r | 0.70 | — | 虚高，无意义 |
| 域对齐（Silhouette） | **0.037** | 高（未测） | ✅ 对齐成功 |
| DEG Pearson（p-value 阈值） | -0.035 | -0.021 | ≈随机 |
| DES 重叠率 | 0.0054 | 0.0042 | 略高于随机 |
| **Top-k PCC（k=3~6）** | **+0.096~+0.126** | -0.11~-0.02 | ✅ MMD 有真实信号 |
| 正相关扰动比例 | **~60%** | ~45–50% | ✅ MMD 高于随机 |

### 5.2 修正后的结论

**旧结论（基于第三轮）**：模型对扰动方向完全没有预测能力，DEG Pearson ≈ 0。

**新结论（基于第六轮）**：

> STATE+MMD 对**最显著的少数基因**（|Wilcoxon scores| 最高的 top-3 到 top-6）有真实的方向预测能力，mean PCC ≈ 0.10~0.13，60% 的扰动能预测正确方向。这个信号在加入更多弱 DEG 时被稀释，导致原方法（p-value 阈值）测不出来。

**Baseline vs STATE+MMD**：

- Baseline 在 k=3 时 mean PCC = **-0.112**，说明没有 MMD 对齐时模型在最显著基因上方向是错的
- STATE+MMD 在 k=3 时 mean PCC = **+0.096**，MMD 对齐确实帮助了扰动方向预测，不只是特征空间的几何对齐

### 5.3 热图解读

- **STATE+MMD 热图**：上半段（约前 30 个扰动）在 top-3 到 top-6 持续红色，说明这些扰动的方向被持续预测正确；下半段偏蓝，说明约一半扰动预测方向错误
- **Baseline 热图**：颜色更混乱，红色区域在 top-3 后快速消失，缺乏一致性
- **两图共同点**：top-2 列颜色最深（±1 artifact）；随着 k 增大颜色逐渐变浅，说明信号随着加入弱 DEG 被稀释

### 5.4 仍然存在的问题

1. **约 40% 的扰动方向预测是错的**：即使在 STATE+MMD 中，仍有相当比例扰动 PCC < 0
2. **预测信号集中在少数基因**：k=3~6 最强，k>7 后 PCC 显著下降，说明对弱 DEG 的方向预测能力很弱
3. **logFC_loss 系数太小（0.1）**：路线一添加的方向损失被 TabularLoss 淹没，对最终结果贡献有限
4. **TabularLoss 本质局限**：Energy Distance 优化分布形状，不奖励逐基因的方向正确性

## 六、根本诊断

```
TabularLoss = Energy Distance（分布级别的统计相似性）
      ↓ 模型的最优策略
预测"分布形状合理"（大多数基因≈0，少数有变化）
      ↓ 导致
对强 DEG 有部分方向信号（top-k PCC > 0）
对弱 DEG 无方向信号（PCC ≈ 0 或负）
      ↓ 整体
DEG Pearson（p-value 阈值，包含弱 DEG）≈ 0
Top-k PCC（只看最强 DEG）≈ 0.10~0.13
```

**域对齐与扰动方向预测是两个独立维度：**

| 维度 | 结果 | 原因 |
|------|------|------|
| 域对齐（CLS embedding 层面） | ✅ 成功 | MMD + GRL 直接优化 CLS 分布距离 |
| 扰动方向预测（binary_decoder 层面） | ⚠️ 部分成功 | TabularLoss 不直接优化方向，但 MMD 对齐间接改善了最强 DEG 的预测 |

## 七、下一步规划

### 现状起点

STATE+MMD 在 top-k 强 DEG 上已经有约 0.10 的 PCC，这是有真实信号的基础，而不是从零开始。

### 方向 1：提高 logFC Loss 权重（1~2 天，低风险）

**动机**：路线一已经加了 logFC MSE，但系数 0.1 太小被 TabularLoss 淹没。

```python
# 当前：0.1 × logfc_loss
# 改为：0.5 或 1.0 × logfc_loss
logfc_weight: 0.5   # 在 config 里调整
```

**验证目标**：top-k PCC（k=5）从 0.126 提升到 > 0.20

**风险**：logfc_loss 量级如果远小于 TabularLoss，即使系数 ×5 也可能效果有限。建议同时记录训练 log 中 `trainer/logfc_loss` 的实际数值。

---

### 方向 2：改损失函数主目标为 logFC MSE（3~5 天，中风险）

**动机**：TabularLoss 是根本瓶颈，不换它就无法突破当前上限。

```
当前：binary_decoder → TabularLoss（分布相似性）
改为：binary_decoder → MSE(pred_logFC, true_logFC)（方向监督）
```

**参考**：GEARS、CPA 均以 logFC 为直接回归目标。

**需要改动：**
1. `DataLoader`：提供每个扰动的 ctrl 均值（或在 batch 里同时包含 ctrl 细胞）
2. `model.py/shared_step`：把 `TabularLoss` 改为（或加权叠加）MSE loss
3. **重新训练**若干 epoch（至少 5~10 epoch 热身）

**验证目标**：top-k PCC（k=5）从 0.126 提升到 > 0.20，最好能在 LOO 实验中也 > 0

---

### 方向 3：换用专门的扰动预测架构（1~2 周，高风险高收益）

**动机**：STATE 的 binary_decoder 设计目标不是预测扰动方向，硬改可能有结构性限制。

**思路**：把 MMD 对齐后的 CLS embedding 作为输入，接 CPA（Compositional Perturbation Autoencoder）或 GEARS 的预测头。

**优点**：CPA/GEARS 的回归头天然为 logFC 预测设计，有很强的方向监督。

**缺点**：工程量最大，需要重新设计数据流和训练流程。

---

### 推荐执行顺序

```
Step 1（本周）：先做方向 1（调高 logfc_weight 到 0.5/1.0）
               → 如果 top-k PCC 明显提升，说明方向损失有效
               → 如果提升有限，直接进入方向 2

Step 2（下周）：方向 2（改主损失为 logFC MSE）
               → 重训练 + top-k PCC 评估
               → 目标：k=5 时 mean PCC > 0.20

Step 3（按需）：如果方向 2 效果仍有限，考虑方向 3（接 CPA/GEARS 头）
```

## Slide 2 — 评估问题的起点：为什么之前的指标失效了

### 内容

**之前做了什么（第一轮）：全基因组 Pearson r**

```
对 HepG2 全部 18,080 个基因：
pearsonr(pred_expression, actual_expression) = 0.70
```

**老师的反馈：这是误导性指标**

```
18,080 个基因的构成：
  ~17,800 个不变基因  ← pred ≈ 0, actual ≈ 0  → 贡献大量"0 vs 0"相关性
  ~280  个 DEG        ← 真正因扰动而变化
```

→ 即使模型什么都没学，只要把所有基因预测为 0，Pearson r 依然虚高

**改进（第三轮）：DEG Pearson（谢老师方法）**

只在 Wilcoxon + BH 校正筛出的 DEG 上算 PCC，STATE+MMD 均值 = **-0.035 ≈ 0**

### 讲稿

> 评估之路从一个被老师否定的指标开始。最初我们在全部 18,000 个基因上计算预测值和真实值的 Pearson r，得到 0.70，看起来不错。但老师指出这是误导性指标——因为这 18,000 个基因里，绝大多数在扰动前后根本没有变化，预测值约等于 0，真实值也约等于 0，这些"0 对 0"的基因贡献了大量虚假的相关性。
>
> 随后我们改用谢老师提出的方法：先用 Wilcoxon 检验 + BH 多重校正找出真正发生变化的差异表达基因，只在这些基因上计算预测方向和真实方向的 Pearson r。结果是 -0.035，约等于 0，看起来模型对扰动方向完全没有预测能力。
>
> 但这个结论是对的吗？今天我想从这里重新出发，深入分析这个评估本身的问题。

### 老师可能的问题

**Q1：BH 校正是什么？为什么要用它？**

> BH（Benjamini-Hochberg）校正是控制多重检验假发现率（FDR）的统计方法。当我们同时对 18,080 个基因做检验时，如果每个基因的显著性阈值设为 p < 0.05，纯粹因为随机波动就会有大约 904 个基因被误判为显著（18,080 × 0.05）。BH 校正通过按 p 值排序后动态调整每个基因的阈值，将整体假发现率控制在指定水平（我们用 adj p < 0.20，即允许最多 20% 的"显著"基因是假阳性）。

**Q2：为什么选 adj p < 0.20 而不是更严格的 0.05？**

> HepG2 的每种扰动细胞数量较少（几十到几百个），样本量小导致检验功效（statistical power）偏低，0.05 的阈值会筛出太少的 DEG，不足以做 Pearson r 计算。0.20 是在"减少假阳性"和"保留足够基因数"之间的权衡。当然，用 0.20 意味着被选入的基因中有更多弱信号基因，这也是原方法信号被稀释的原因之一。

## Slide 3 — 原方法的根本问题：弱 DEG 稀释了信号

### 内容

**BH 阈值筛出的 DEG 构成（示意）：**

```
每个扰动筛出的 DEG（adj p < 0.05）：
┌────────────────────────────────────────────┐
│ top few 基因：|logFC| 大，方向明确         │ ← 真实信号
│ 中间大部分：  |logFC| 中等，方向模糊       │ ← 弱信号
│ 末尾若干：    adj p 刚好通过阈值，|logFC|≈0│ ← 噪声
└────────────────────────────────────────────┘

Pearson r 是对所有 DEG 一视同仁地平均
→ 少数强信号被大量弱信号/噪声稀释
→ mean PCC → 0 甚至负值
```

**问题不是"模型没有信号"，而是"评估方法没有放大信号"**

### 讲稿

> DEG Pearson 方法是正确的思路，但在执行层面有一个隐藏问题：BH 校正筛出的 DEG 里，并不是每个基因的差异都同样显著。排名靠前的基因差异非常大，方向明确；排名靠后的基因勉强通过统计阈值，logFC 很小，方向实际上接近噪声。
>
> Pearson r 的计算是对所有被选中的基因一视同仁——它不管这个基因的差异有多大，只看预测方向和真实方向是否一致。这样，少数几个方向预测准确的强差异基因，就被大量方向不确定的弱差异基因的随机噪声淹没了。
>
> 所以我们的问题是：**能不能设计一种评估方法，专门聚焦在差异最显著的基因上，看模型在那里有没有真实的预测能力？**

### 老师可能的问题

**Q3：那你的新方法是不是也有问题——只看几个基因，样本太少，Pearson r 本身就不稳定？**

> 对，这是一个真实的 trade-off。k=3 时每个扰动只有 3 个 (pred_logFC, actual_logFC) 数据对，Pearson r 的方差很大（我们看到 std ≈ 0.70）。但我们不是只看单个扰动的 Pearson r，而是对 67 个扰动取均值——平均之后 mean 的方差是 0.70/√67 ≈ 0.085，mean PCC = 0.10 对应约 1.2 个标准误，虽然不够强，但方向是一致正的，而 Baseline 同样 k=3 的 mean PCC = -0.11，两者差距约 0.21，差距超过 2 个标准误。这是有统计意义的信号，不是随机波动。

## Slide 4 — 新方法核心：Wilcoxon Scores 是什么

### 内容

**scanpy `rank_genes_groups` 返回的 `scores` 列**

```python
sc.tl.rank_genes_groups(
    adata_combined,
    groupby="_group",
    groups=["perturbed"],    # 扰动组
    reference="control",     # 对照组
    method="wilcoxon",
    corr_method="benjamini-hochberg",
)
de_df = sc.get.rank_genes_groups_df(adata_combined, group="perturbed")
# 列：names | scores | logfoldchanges | pvals | pvals_adj
```

**`scores` 的本质：Wilcoxon 秩和统计量的标准化值（z-score）**

```
对每个基因 g：
  将扰动组 + 对照组的所有细胞合并，对基因 g 的表达值排秩
  U = 扰动组细胞在总秩中的秩和
  z = (U - μ_U) / σ_U    ← 标准正态近似
  
  scores[g] = z
```

**正负含义：**

| scores 符号 | 实际含义 |
|-------------|---------|
| **scores > 0** | 扰动组基因 g 的秩 **高于** 对照组 → 扰动后**上调** |
| **scores < 0** | 扰动组基因 g 的秩 **低于** 对照组 → 扰动后**下调** |
| scores ≈ 0 | 两组无差异 |

**|scores| 的含义：** 差异的显著程度（同时综合效应量和样本量）

### 讲稿

> 我们的新方法建立在 scanpy 的 `rank_genes_groups` 函数上。这个函数对每个基因做 Wilcoxon 秩和检验，返回一个 `scores` 列，这是我们方法的核心输入。
>
> Wilcoxon 秩和检验的做法是：把扰动组和对照组的所有细胞合并，对每个基因的表达量排秩——表达量最低的细胞排第 1，最高的排最后。然后统计扰动组细胞的秩和 U。如果扰动组的表达量普遍高于对照组，U 就会偏大；如果普遍低，U 就偏小。这个秩和经过期望和标准差的标准化，得到近似服从标准正态分布的 z 统计量，就是 `scores`。
>
> 关键点在于符号：scores > 0 表示扰动组的表达量整体高于对照组，也就是该基因在扰动后上调；scores < 0 表示下调。这个符号和实际 logFC 的正负是一致的。绝对值 |scores| 反映差异的显著程度——越大说明差异越稳定、越不像随机波动。

### 老师可能的问题

**Q4：Wilcoxon 检验是非参数方法，和 t 检验有什么区别？为什么用 Wilcoxon 而不是 t 检验？**

> t 检验假设数据服从正态分布，对基因表达数据（log1p 归一化后仍然是右偏的计数分布）这个假设不成立。Wilcoxon 秩和检验是非参数方法，不对数据分布做任何假设，只用秩信息，因此对基因表达数据更鲁棒。scanpy 默认推荐 Wilcoxon 方法，也是单细胞领域的主流选择。

**Q5：scores 和 logfoldchanges 有什么关系？能相互替代吗？**

> 两者都反映扰动方向，符号方向是一致的（scores > 0 对应 logFC > 0）。区别在于：logFC 是纯效应量指标，只看均值之比，不管方差多大；scores 是统计量，综合了效应量和样本量（细胞数多时同样的 logFC 会产生更大的 |scores|）。用 |scores| 排序比用 |logFC| 更鲁棒——当细胞数很少时，单个细胞的随机波动可以造成虚大的 logFC，但检验功效不足，scores 会维持在较小值，不会被错误地排到前面。

**Q6：scores 可以直接用来做 PCC 计算吗（替代 logFC）？**

> 理论上可以，scores 的符号方向和 logFC 一致。但 scores 的量纲是标准化的统计量，不是基因表达的自然单位，且 scores 的绝对值受样本量影响（细胞数多时同样效应的 scores 绝对值更大）。我们选择用 logFC 做 PCC 计算——因为 PCC 是无量纲的相关系数，不受 logFC 绝对量级影响，只看方向和相对大小关系，这正是我们想测量的。用 scores 排序（选哪些基因），用 logFC 计算 PCC（方向是否预测正确），两者分工明确。

## Slide 5 — 为什么用 |scores| 排序，而不是 p 值或 |logFC|

### 内容

**三种排序方式的对比：**

| 排序依据 | 优点 | 缺点 |
|---------|------|------|
| adj p 值（由小到大）| 控制假阳性 | 细胞数多时很多基因都显著，无法区分强弱 |
| \|logFC\|（由大到小）| 直觉明确 | 样本量小时 logFC 虚大，不稳定 |
| **\|scores\|（由大到小）** | 综合效应量+统计检验力，稳定 | 受细胞数影响（但这反而是优点） |

**|scores| 排序的直觉：**

```
|scores| 大 = 这个基因的差异在统计上非常确定
           = 我们可以相信它的 logFC 方向是真实的
           = 是评估模型"应该最容易预测对"的基因
```

**我们的思路：**

> 如果模型有任何方向预测能力，应该首先体现在差异最显著的基因上。
> 用 |scores| 最大的 top-k 基因是"信号最强的测试集"。

### 讲稿

> 选什么基因来计算 PCC，是这个方法设计的核心决策。
>
> 用 adj p 值筛基因（原方法）的问题是：当细胞数足够多时，很多基因都能通过显著性阈值，但通过了阈值不代表差异大，只代表差异可检测。BH 阈值筛出的基因里良莠不齐，最弱的基因 logFC 只有 0.01，方向性几乎是噪声。
>
> 用 |logFC| 排序的问题是：当某个扰动只有几十个细胞时，单个细胞的随机波动就能造成虚大的 logFC，排在前面的未必是真正差异稳定的基因。
>
> 用 |scores| 排序是最鲁棒的选择。scores 是 Wilcoxon 统计量的标准化，它既考虑效应量（logFC 越大，秩差越大），也考虑统计检验力（细胞数越多，同样的效应量产生更大的 |scores|）。|scores| 最大的基因，是"在这么多细胞里统计上最确定地发生了差异"的基因——也是我们最有理由相信其 logFC 方向是真实的基因，因此是评估模型预测能力的最佳测试集。

### 老师可能的问题

**Q7：你用 |scores| 排序选出的基因，是不是和用 p 值排序选出的基因几乎一样？**

> 在细胞数相同的情况下，两者高度相关（Wilcoxon 统计量 U 直接决定 p 值，|scores| 越大 p 值越小）。但当扰动组细胞数差异很大时（比如有的扰动有 200 个细胞，有的只有 20 个），|scores| 会纳入细胞数的影响，p 值排序也会体现这一点，但直接用 |logFC| 排序就会出现偏差。在实践中，|scores| 和 p 值排序在 top-k 的选择上基本一致，主要差别在于 |scores| 提供了一个连续值，而 p 值在统计截断处会有跳跃。

## Slide 6 — 方法流程：Top-k PCC 曲线的完整计算

### 内容

**对每个扰动（如 TP53 敲除），执行四步：**

**Step 1：Wilcoxon 排序（用真实数据）**
```
扰动组细胞 + 对照组细胞
       ↓ sc.tl.rank_genes_groups (Wilcoxon)
scores[g] 有正负，|scores[g]| 从大到小排列 18,080 个基因
→ ranked_genes = [基因1, 基因2, ..., 基因18080]
```

**Step 2：模型推理（用模型预测）**
```
对照组细胞 → Transformer → 平均 CLS → binary_decoder → pred_ctrl[18080]
扰动组细胞 → Transformer → 平均 CLS → binary_decoder → pred_pert[18080]

pred_logfc_all   = pred_pert   - pred_ctrl    （预测 logFC，全部 18080 基因）
actual_logfc_all = actual_pert - actual_ctrl  （真实 logFC，全部 18080 基因）
```

**Step 3：for k = 1, 2, ..., 10**
```
genes_k = ranked_genes[:k]          ← 前 k 个基因
idx_k   = 这些基因在 18080 中的下标

PCC_k = pearsonr(pred_logfc_all[idx_k],
                  actual_logfc_all[idx_k])
```

**Step 4：汇总 67 个扰动**
```
mean_PCC(k) = mean([PCC_k(TP53), PCC_k(MED1), ..., PCC_k(HDAC1)])
→ 得到一条曲线：{k: mean_PCC(k)}
```

### 讲稿

> 方法流程分四步，前两步完全独立——排序用真实数据，推理用模型，两者不互相依赖，不存在信息泄露。
>
> 第一步，对每个扰动用 Wilcoxon 检验对全部 18,080 个基因排序，按 |scores| 从大到小，排名第一的是该扰动下差异最显著的基因。
>
> 第二步，用模型分别编码对照组和扰动组细胞，取平均 CLS embedding 后解码出全部基因的预测分数，差值就是模型预测的 logFC。
>
> 第三步是核心：对 k 从 1 增大到 10，每次取前 k 个基因的 predicted logFC 和 actual logFC，算一次 Pearson r。这给了我们一张"k 个基因时模型有多准"的快照。
>
> 第四步，对 67 个扰动的 PCC 取均值，画成曲线。曲线的形状本身就是信息——如果曲线在小 k 时高、随 k 增大而下降，说明模型的信号集中在最显著的基因上；如果曲线平坦，说明信号均匀分布。

### 老师可能的问题

**Q8：Step 1 和 Step 2 都用了真实数据，有没有数据污染的问题？**

> 没有。Step 1（Wilcoxon 排序）用真实数据决定"选哪些基因"，Step 2 的模型推理完全不看真实标签——模型只输入细胞的基因表达矩阵，输出预测的 logFC。然后 Step 3 用真实 logFC 和预测 logFC 做比较。这与有监督学习中用测试集标签评估模型是完全一样的逻辑，是标准做法，不构成数据泄露。

**Q9：为什么 k 最多取到 10？为什么不取更大？**

> 我们设 top_k_max=10 是为了专注于"差异最显著的极少数基因"。如果 k 很大（比如 100），实际上又退化成了加入大量弱 DEG 的情况，和原方法的问题相同。从结果也可以看到，k=7 以后 PCC 就快速下降了，说明第 7~10 个基因对 PCC 的贡献已经是稀释而非增强。我们计划在后续实验中也尝试更大的 top_k_max（如 20 或 50），画出完整的衰减曲线，确认信号衰减的速度。

## Slide 7 — 特殊情况：k=1 和 k=2 为什么不可用

### 内容

**k=1：NaN（无法计算）**

```python
pearsonr([pred_logfc_gene1], [actual_logfc_gene1])
# 只有 1 个数据点，Pearson r 无法定义
# → 输出 NaN，从均值计算中排除
```

Pearson r 衡量的是两组数列的线性相关性，最少需要 2 个数据点才能定义直线。

**k=2：±1 的数学 artifact**

```python
pearsonr([pred_A, pred_B], [actual_A, actual_B])
# 2 个点必然落在同一条直线上（两点确定一条直线）
# → pearsonr 恒等于 +1 或 -1，不代表预测准确
```

**k=2 结果（仅供参考）：**

| 模型 | mean PCC at k=2 | 等价含义 |
|------|----------------|---------|
| STATE+MMD | 0.134 | 67 个扰动中，约 57% 的扰动"最显著基因和次显著基因的方向一致" |
| Baseline | 0.015 | 约 51%，接近随机 |

**结论：实际分析从 k=3 开始。**

### 讲稿

> 在解读结果之前，必须先说清楚 k=1 和 k=2 为什么不能直接用。
>
> k=1 时，每个扰动只有一个数据点（一个基因的 predicted logFC 和 actual logFC），Pearson r 需要至少两个数据点才有意义，所以输出 NaN，从均值计算中自动排除。
>
> k=2 时有一个更微妙的数学问题：两个点永远落在同一条直线上，两点确定一条直线，而 Pearson r 衡量的就是点和最优直线的拟合程度，所以结果恒等于 +1 或 -1。这不代表模型预测准确，只代表两个点的方向是否一致。STATE+MMD 在 k=2 的均值是 0.134，等价于：在 67 个扰动中，57% 的扰动里最显著的基因和次显著的基因预测方向一致，Baseline 是 51%，接近随机。这个信息有一定参考价值，但不是我们的核心指标。
>
> 因此，今天的核心结论全部基于 k=3 到 k=10。

### 老师可能的问题

**Q10：既然 k=2 总是 ±1，那表里显示的 mean=0.134 是怎么来的？**

> 因为 67 个扰动里，每个扰动的 k=2 结果要么是 +1、要么是 -1。均值 0.134 = (n_plus - n_minus) / 67，其中 n_plus 是结果为 +1 的扰动数，n_minus 是结果为 -1 的扰动数。STATE+MMD 的 (57×1 + 10×(-1)) / 67 ≈ 0.134，说明有约 57 个扰动得到 +1、10 个得到 -1。Baseline 的 0.015 约等于有 34 个 +1、33 个 -1，接近随机翻硬币。

> 这张图是今天结果的核心。横轴是 k（我们选了几个基因来算 PCC），纵轴是 67 个扰动的 mean PCC。
>
> 看红线（STATE+MMD）：从 k=3 开始，mean PCC 约 0.10，在 k=5 时达到峰值 0.126，之后随着加入更多基因开始缓慢下降，在 k=10 时降到 0.053。全程在 0 以上。
>
> 看蓝线（Baseline，无 MMD 对齐）：k=3 时 mean PCC = -0.112，是负的，说明 Baseline 在最显著的 3 个基因上方向预测是系统性偏反的。之后随 k 增大慢慢回到 0 附近。
>
> 阴影区域是 ±1 标准差，很宽，说明不同扰动之间差异极大——有些扰动 PCC 接近 +1，有些接近 -1。这很正常，我们感兴趣的是 67 个扰动的均值趋势。
>
> 红线全程高于蓝线，在 k=3~7 的区间差距最大（约 0.10~0.20）。这说明 MMD 域对齐不只是在特征空间层面把三个细胞系融合了，它对 binary_decoder 的方向预测能力也有实质性的正向贡献。

### 老师可能的问题

**Q11：std 那么大（0.7），mean=0.10 有统计意义吗？**

> 这是一个合理的质疑。single-perturbation 的 PCC 方差很大，是因为 k=3 时只有 3 个数据点，pearsonr 在小样本下的方差天然就大。但我们计算的是 67 个扰动的 mean，mean 的标准误 = std / √67 ≈ 0.70 / √67 ≈ 0.085。STATE+MMD 的 mean = 0.096，约是 1.1 个标准误，单模型的显著性偏弱。但两模型的差值 ≈ 0.208，差值的标准误 ≈ √(2) × 0.085 ≈ 0.12，差值约是 1.7 个标准误，p ≈ 0.09，在 10% 水平上显著。更重要的是，这个方向（MMD > Baseline）在 k=3~10 的所有 k 上都一致，不是偶然的。我们下一步会计划用置换检验（permutation test）来量化显著性。

**Q12：mean PCC = 0.10 在生物学上有意义吗？这个数字够不够好？**

> 对于单细胞扰动预测，DEG Pearson 在 0.10~0.20 是已有工作报告的一个常见区间（GEARS 在某些数据集上约 0.15~0.30，但用的是完整监督的 logFC MSE 训练）。我们的 0.10 来自一个**未针对扰动方向优化损失函数**的模型，只是 TabularLoss + 较小权重的 logFC_loss，能达到 0.10 说明 MMD 对齐确实捕捉到了部分扰动相关信息。但 0.10 距离实用标准（> 0.30）还有差距，这也是我们下一步要改进的方向。

### 老师可能的问题

**Q13：Baseline 在 k=3 的方向为什么是反的（PCC = -0.11）？有什么机制上的解释？**

> 这个问题很好，我们目前的推测是：没有 MMD 对齐时，三个训练域（K562/RPE1/Jurkat）的 CLS embedding 在特征空间中是分开的。当模型在 HepG2（从未见过的细胞系）上推理时，HepG2 的 CLS embedding 可能落在训练域分布的边界区域，binary_decoder 在解码时可能出现系统性偏差——对某类基因（可能是 HepG2 的细胞系特异性高表达基因）的方向判断发生了反转。MMD 对齐把三个域融合后，HepG2 在特征空间里有了更合适的"落脚点"，这个系统性偏差被纠正了。但这只是推测，要确认需要进一步分析哪些基因在 Baseline 上方向反了，看它们有没有共同的生物学特征。

## Slide 11 — 修正后的结论：信号存在，但被原方法稀释了

### 内容

**原结论（基于 DEG Pearson，p-value 阈值）：**

```
STATE+MMD mean PCC = -0.035 ≈ 0
→ 模型对扰动方向没有预测能力
```

**新结论（基于 Top-k PCC 曲线）：**

```
STATE+MMD mean PCC (k=3~6) ≈ +0.10~+0.13
→ 模型对最显著差异基因有真实的方向预测能力
```

**为什么两种方法得出截然不同的结论：**

```
原方法（p-value 阈值）：
  每个扰动纳入 30~200 个 DEG
  其中：前 3~6 个有信号（PCC ≈ +0.10）
        后面 24~194 个是噪声（PCC ≈ 0 或随机）
  平均 → PCC → 接近 0，甚至负

Top-k 方法：
  只取前 3~6 个，信号不被稀释
  → PCC ≈ +0.10~+0.13
```

**两个发现都是真实的，说的是同一件事的两面：**

> 模型的方向预测信号是稀疏的——集中在少数最显著的基因上，对弱 DEG 没有预测能力。

### 讲稿

> 这里要强调一件事：这两个结论不矛盾，说的是同一现象的两个层次。
>
> "DEG Pearson（p-value 阈值）≈ 0"告诉我们：如果把 BH 校正筛出的全部 DEG 都纳入，平均下来模型的方向预测就是随机水平。这是真的。
>
> "Top-k PCC ≈ 0.10"告诉我们：如果只看差异最显著的 3~6 个基因，模型的方向预测好于随机。这也是真的。
>
> 两者放在一起告诉我们：**模型的方向预测信号是稀疏的**——只集中在最强的极少数基因上，对弱 DEG 的方向几乎没有预测能力。原方法把强信号和大量噪声平均，所以信号消失了；新方法聚焦在强信号，才把它暴露出来。

### 老师可能的问题

**Q16：你这个分析是不是有"事后选择"（post-hoc selection）的问题？你看了结果之后才设计了一个能得到正面结论的评估方法？**

> 这是最重要的一个问题，必须正面回答。这个设计思路不是在看到 -0.035 之后才有的，而是来自对原方法缺陷的先验分析：p-value 阈值方法包含弱 DEG 是一个已知问题，不是结果出来之后的反应。但我承认，最终选择 k=3~10 这个范围，是在看到结果之后确定的，k=2 是 artifact、k>7 PCC 开始下降，这些都是结果驱动的选择。
>
> 更好的做法是在独立数据集上验证——如果在 K562 或 RPE1（训练域）上也能看到同样的 top-k PCC 曲线形状，会大大增加可信度。我们下一步会做这个验证。在此之前，这个结论应该被标注为"探索性发现"而不是确认性结论。

                               → 对弱 DEG 完全没有方向约束（PCC ≈ 0）

MMD 对齐 → CLS embedding 融合 → 最强 DEG 的方向偏差被纠正
         → 不解决 TabularLoss 对弱 DEG 无监督的问题

>

## Slide 13 — 下一步：三条方向

### 内容

**现状**：STATE+MMD 在 top-k 强 DEG 上已有约 0.10 的 PCC，这是有真实信号的基础。

**方向 1（最快，1~2 天）：提高 logFC Loss 权重**

```yaml
# 当前 config：
logfc_weight: 0.1   # 太小，被 TabularLoss 淹没

# 改为：
logfc_weight: 0.5   # 或 1.0
```

验证目标：top-k PCC（k=5）从 0.126 → > 0.20

**方向 2（中等，3~5 天）：改主损失为 logFC MSE**

```
当前：binary_decoder → TabularLoss（分布相似性）
改为：binary_decoder → MSE(pred_logFC, true_logFC)
参考：GEARS、CPA 的标准做法
```

验证目标：top-k PCC > 0.20，LOO 实验中训练域 PCC > 0

**方向 3（彻底，1~2 周）：换用专门架构**

将 MMD 对齐后的 embedding 接 CPA/GEARS 预测头。

### 讲稿

> 诊断清楚了，下一步是修复。根本问题是 TabularLoss 的优化目标和方向预测的目标不匹配。
>
> 最快的尝试是方向一：提高 logfc_loss 的权重。路线一已经在 shared_step 里加了 logFC MSE，但系数只有 0.1，被 TabularLoss 淹没了。把权重提到 0.5 或 1.0，重新训练几个 epoch，看 top-k PCC 曲线是否上升。这是改动最小、最容易验证的实验。
>
> 如果方向一不够，就进入方向二：改主损失。当前的 TabularLoss 完全换掉，或者加大 logFC MSE 的比重，让方向监督成为主导梯度信号。这需要改 DataLoader 来在训练时提供 ctrl 均值，以及调整 model.py 的 shared_step。
>
> 方向三是最彻底的：保留 MMD 对齐作为 embedding 前处理，但把预测头换成 CPA 或 GEARS，这两个架构天然以 logFC 为回归目标，方向监督非常强。工程量最大，但最有把握达到 DEG Pearson > 0.30 的实用水平。

### 老师可能的问题

**Q17：你为什么不先做方向二，而是先做方向一？**

> 方向一的成本极低——只需改一行 config 的数字，在服务器上微调几个 epoch，然后用同样的评估脚本出结果，1~2 天能看到答案。如果方向一有效（top-k PCC 明显提升），说明 logFC_loss 的方向是对的，只是权重问题；如果方向一无效（PCC 基本不变），则说明 logFC_loss 的量级相对于 TabularLoss 太小，权重再大也会被淹没，那就必须从根本上替换损失函数（方向二）。方向一是一个低成本的探针实验，它的结果直接决定方向二是否必要。

**Q18：top-k PCC > 0.20 这个验证目标是怎么定的？**

> 这个目标是相对于当前 0.126 的提升目标（约提升 60%），以及我们认为"有方向预测能力"的最低门槛。文献中（GEARS 等）DEG Pearson > 0.20 通常被认为模型具有一定的扰动预测能力。当然，这个数字是参考性的，不是绝对的判断标准。更重要的是方向：任何明显的提升都值得继续深入。

## 附录：关键公式速查

### Wilcoxon 秩和统计量

```
合并扰动组和对照组，对基因 g 的表达值排秩：
U = Σ(扰动组细胞的秩) - n_pert × (n_pert + 1) / 2

期望：μ_U = n_ctrl × n_pert / 2
方差：σ²_U = n_ctrl × n_pert × (n_ctrl + n_pert + 1) / 12  （含平局修正）

z = (U - μ_U) / σ_U  ← 即 scores
```

### Pearson 相关系数

```
PCC = Σ[(pred_i - pred̄)(actual_i - actual̄)] /
      (√Σ(pred_i - pred̄)² × √Σ(actual_i - actual̄)²)

其中：i = 1..k（top-k 基因）
     pred_i = pred_logFC[gene_i] = decode(CLS_pert)[i] - decode(CLS_ctrl)[i]
     actual_i = actual_logFC[gene_i] = mean_pert_expr[i] - mean_ctrl_expr[i]
```

### BH 校正

```
将 m=18,080 个基因的 p 值从小到大排序为 p_(1) ≤ p_(2) ≤ ... ≤ p_(m)
找最大的 k 使得 p_(k) ≤ k × q / m  （q = FDR 阈值，我们用 q=0.20）
前 k 个基因被判定为显著
```

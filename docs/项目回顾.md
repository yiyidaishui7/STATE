# 全项目回顾

---

## 背景：做的是什么

用 MMD-AAE（最大均值差异 + 对抗自编码器）对 STATE 预训练模型做**域对齐**，目标是让 K562/RPE1/Jurkat 三个细胞系的 CLS embedding 占据同一区域，使得 HepG2（从未见过的细胞系）能零样本迁移。

---

## 已完成的五轮实验

### 第一轮：全基因组 Pearson r（eval_hepg2_pearson.py）

- **做了什么**：在 HepG2 全部 ~18,000 个基因上计算预测值 vs 真实值的 Pearson r
- **结果**：r ≈ 0.70
- **问题**：被老师指出是误导性指标——大量不变基因（predicted≈0, true≈0）把相关系数虚高了，即使模型什么都没学会也能拿到 0.7

### 第二轮：域对齐质量评估（eval_state_domain_alignment.py）

- **做了什么**：定量检验 MMD-AAE 的对齐效果，用四个指标：域分类准确率、Silhouette Score、MMD²、CORAL
- **结果**：DomainClsAcc=99.61%，Silhouette=0.037≈0，MMD²=6e-4，CORAL=0
- **结论**：域对齐在表征空间层面是成功的，三个细胞系的 embedding 确实融合了

### 第三轮：DEG Pearson（eval_deg_pearson.py，谢老师方法）

- **做了什么**：对每个扰动，先用 Wilcoxon+BH 筛出显著差异表达基因（DEG），只在这些 DEG 上算预测 log-FC vs 真实 log-FC 的 Pearson r
- **结果**：HepG2 STATE+MMD mean=-0.035，Baseline mean=-0.021，两者均≈0
- **结论**：在真正发生变化的基因上，预测完全没有方向性

#### 第三轮详解

**为什么要做这个实验**

第一轮全基因组 Pearson r = 0.70 被老师否定。原因：18,000 个基因里只有少数几百个真的因敲除而变化，其余大多数基因预测值≈0、真实值也≈0，这些"不变基因"把相关性虚高了。老师要求的解法：先找出真正发生变化的基因，只在这些基因上算相关性。

**数据准备**

HepG2 数据（`hepg2.h5`）里每个细胞有一个 `gene` 标签（被敲除的基因名），对照组标签是 `"non-targeting"`。X 是 log1p 归一化后的基因表达矩阵，形状 `(n_cells, 18080)`。

```
HepG2 数据：
  ~5000 个对照细胞（non-targeting）
  68 种敲除扰动，每种约几十到几百个细胞
  18080 个基因
```

**对每个扰动做的四件事**

以敲除 TP53 为例：

---

**第一件事：找真实 DEG（Wilcoxon + BH 校正）**

```python
adata_ctrl.obs["_group"] = "control"
adata_pert.obs["_group"] = "perturbed"
adata_combined = concat([adata_ctrl, adata_pert])

sc.tl.rank_genes_groups(
    adata_combined,
    groupby="_group",
    groups=["perturbed"],
    reference="control",
    method="wilcoxon",
    corr_method="benjamini-hochberg",
)
de_df = sc.get.rank_genes_groups_df(..., pval_cutoff=0.20)
```

- **Wilcoxon 检验**：对每个基因，把扰动组和对照组的细胞表达值做非参数秩和检验，看两组分布是否有显著差异
- **BH 校正**：同时检验 18,000 个基因会产生大量假阳性，BH 校正控制整体假发现率（FDR）
- 结果：TP53 敲除后有统计显著差异的基因，例如筛出 30~200 个 DEG

---

**第二件事：计算这些 DEG 的真实 log-FC**

```python
actual_ctrl_mean = adata.X[ctrl_indices][:, de_gene_indices].mean(axis=0)
actual_pert_mean = adata.X[pert_indices][:, de_gene_indices].mean(axis=0)
actual_logfc = actual_pert_mean - actual_ctrl_mean   # shape: (n_de_genes,)
```

每个 DEG 得到一个真实 log-FC 值，正数表示敲除后该基因上调，负数表示下调。

---

**第三件事：用模型预测这些基因的变化**

两步走：

**Step A：把细胞编码成 CLS embedding（512维）**

```python
# 对照组 200 个细胞 → Transformer → CLS token → 取均值
ctrl_cls_mean = encode_cells_to_cls(model, h5_path, ctrl_indices).mean(axis=0)  # (512,)

# 扰动组做同样操作
pert_cls_mean = encode_cells_to_cls(model, h5_path, pert_indices).mean(axis=0)  # (512,)
```

`encode_cells_to_cls` 内部：把每个细胞的基因表达 token 化为基因句子 → Transformer 前向传播 → 取 `gene_output[:, 0, :]`（CLS 位置）→ decoder → 512 维向量。

**Step B：把 CLS embedding 解码到全部 18,080 个基因的预测分数**

```python
gene_embs = model.get_gene_embedding(gene_names)     # (18080, 512) ESM2蛋白质向量→线性投影

pred_ctrl_scores = predict_from_emb(ctrl_cls_mean, gene_embs)  # (18080,)
pred_pert_scores = predict_from_emb(pert_cls_mean, gene_embs)  # (18080,)
```

`predict_from_emb` 内部（分批处理避免 OOM）：

```
for 每批 512 个基因:
    combine = cat([gene_emb_batch, CLS_mean_expanded, mu])   # (1, 512, 1025)
    scores = binary_decoder(combine)                          # (512,)
```

binary_decoder 输入 = `[基因emb ‖ CLS_emb ‖ mu]`（1025维），输出每个基因一个扰动分数。

**算预测 log-FC**：

```python
pred_logfc = pred_pert_scores[de_gene_indices] - pred_ctrl_scores[de_gene_indices]
# 只取 DEG 子集，shape: (n_de_genes,)
```

---

**第四件事：计算 Pearson r**

```python
r, pval = pearsonr(pred_logfc, actual_logfc)
```

`pred_logfc` 和 `actual_logfc` 各是一个长度为 n_de_genes 的向量（例如 30 个 DEG 各有一个预测值和真实值）。

Pearson r 衡量：**模型预测的变化方向是否与真实方向一致**。

| r 值 | 含义 |
|------|------|
| +1 | 模型完美预测出哪些上调哪些下调 |
| 0 | 预测与真实方向完全无关（随机猜测水平） |
| -1 | 模型系统性预测反了 |

收集所有 65 个扰动的 r 值，取均值 = **-0.035**（STATE+MMD），**-0.021**（Baseline）。

**为什么结果是 ≈ 0**

均值 -0.035，正相关比例 ~48%——这就是随机猜测的水平（期望值 0，正相关比例 50%）。

`binary_decoder` 的训练目标是 TabularLoss（energy distance）：让预测的基因分数**分布**在统计上接近真实分布，不是让每个基因的方向正确。模型学会了"大多数基因变化量接近 0，少数有变化，整体分布合理"就能把 loss 压低，根本不需要学会哪些基因上调哪些下调。全基因组 Pearson r = 0.70 是因为 17,800 个不变基因（predicted≈0, true≈0）贡献了虚高的相关性，掩盖了真正重要的 DEG 上的完全随机预测。

---

### 第四轮：LOO 留一法实验（四细胞系）

- **做了什么**：分别训练四个模型，每次留一个细胞系作为零样本验证，检验 DEG Pearson 是否在训练集细胞系上 > 0
- **结果**：全部四个验证集 DEG Pearson ≈ 0（-0.035 ~ +0.013），正相关比例 45-48%，与随机猜测无异
- **结论**：不是域泛化问题（否则训练集细胞系应该能预测），是架构/损失函数本身的问题——TabularLoss（energy distance）优化的是分布相似性，不奖励扰动方向正确

### 第五轮：DES gene overlap（eval_des.py）

- **做了什么**：STATE 原生的另一个指标——预测 top-50 基因与真实 top-50 DEG 的重叠率（DES = 交集大小 / k）
- **结果**：STATE+MMD=0.0054，Baseline=0.0042，随机基线=0.0028，median=0，DES>0 仅 21%
- **结论**：与 DEG Pearson 完全一致，两个指标从不同角度（方向相关 vs 命中率）都指向同一诊断

---

## 总诊断

```
TabularLoss = energy_distance(预测分布, 真实分布)
```

模型通过预测"接近零、分布合理"的输出就能把这个损失压下去，完全不需要学会"哪些基因上调/哪些下调"。这是损失函数设计的根本局限，与域对齐无关。

---

## 下一步应该做什么

根本问题清楚了，下一步是**换损失函数**，三条可选方向按难度排序：

**方向 1（最快，1-2天）：在现有 loss 上加 Pearson 正则项**

```python
# 在 shared_step 中追加
pearson_loss = -pearson_r(pred_logfc, true_logfc)  # 最大化相关性
total_loss = tabular_loss + λ * pearson_loss
```

- 优点：改动最小，可以直接在现有代码上加
- 缺点：Pearson r 不可微（需要 soft 版本），且 true_logfc 在训练时不一定直接可得

**方向 2（中等，3-5天）：把监督目标换成 log-FC**

```
当前：binary_decoder 输出 → TabularLoss(分布)
改为：binary_decoder 输出 → MSE(预测 log-FC, 真实 log-FC)
```

- 参考 GEARS、CPA 的做法——直接以 log fold-change 为回归目标
- 需要改 DataLoader（提供 ctrl 均值）和损失函数

**方向 3（彻底，1-2周）：换用专门的扰动预测模型**

- 把 MMD 对齐模块接到 CPA 或 GEARS 上游，用对齐后的 embedding 驱动有更强监督的预测头
- 工程量最大，但最有可能真正解决问题

**最推荐的起点**：方向 2——改监督目标为 log-FC MSE，同时保留 MMD 对齐。改动范围可控，逻辑直接，验证目标明确（DEG Pearson > 0.10）。
